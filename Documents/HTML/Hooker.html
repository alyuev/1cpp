предопределенных событий 1С и ВК FormEx, вызываемых для групповых контекстов, модулей проведения документов и/или глобального модуля, с помощью специальных объектов пользовательских классов 1С++ или объектов типа ВыполняемыйМодуль или наследников от произвольных CBL-контекстов.</p> <p><strong>Версия: релиз 2.5 и выше.</strong></p> <div class="section" id="id3"> <h1><a name="id3"><strong>Общая схема перехвата:</strong></a></h1> <chminfo split="0" splitchild="1"> <insert nodes="//common/propsmethods/*"/> </chminfo> <p>0. Для использования перехвата должна быть включена настройка 1С++ " <strong>ПерехватитьСобытияГК / EnableHookEventsGroupContext</strong> " См. <a class="reference" href="SettingsManager.chm::/SettingsManager0.html">SettingsManager</a></p> <p>1.0 В качестве собственных объектов-перехватчиков можно использовать как пользовательский класс 1С++, так и объект типа ВыполняемыйМодуль, а также наследников от произвольных CBL-контекстов (например, для произвольных ВК - подобная ВК со своими классами есть у Александра Орефкова aka orefkov)</p> <p>1.1 Необходимо использовать модуль, в котором должны быть определены <strong>обработчики соответствующих событий с названием события</strong> (ПриЗакрытии, ПриНачалеВыбораЗначения, ПриНажатииКнопкиКлавиатуры) или <strong>со специальным названием в виде Событие_НаименованиеСобытия</strong> (Событие_ПриЗакрытии, Событие_ПриНачалеВыбораЗначения, СобытиеГМ_ПриНажатииКнопкиКлавиатуры)</p> <p>1.2 Для событий глобального модуля <strong>всегда используется префикс ``СобытиеГМ_``</strong> (СобытиеГМ_ПриЗавершенииРаботыСистемы, СобытиеГМ_ПриНажатииКнопкиКлавиатуры)</p> <p>2. Каждый обработчик должен быть определен <strong>с атрибутом Экспорт</strong>, иначе перехват указанного события будет невозможен.</p> <p>Например, для перехвата событий ПриЗакрытии и ПриНачалеВыбораЗначения в классе-перехватчике необходимо написать</p> <blockquote> <dl class="docutils"> <dt>Процедура ПриЗакрытии() Экспорт</dt> <dd>// текст обработчика события</dd> </dl> <p>КонецПроцедуры</p> <dl class="docutils"> <dt>Процедура Событие_ПриНачалеВыбораЗначения(НужныеПараметры и т.д.) Экспорт</dt> <dd>// текст обработчика события</dd> </dl> <p>КонецПроцедуры</p> </blockquote> <p>Или для перехвата события ПриЗавершенииРаботыСистемы в классе-перехватчике необходимо написать</p> <blockquote> <dl class="docutils"> <dt>Процедура СобытиеГМ_ПриЗавершенииРаботыСистемы() Экспорт</dt> <dd>// текст обработчика события</dd> </dl> <p>КонецПроцедуры</p> </blockquote> <p>3. Для включения перехвата конкретного контекста формы нужно использовать код</p> <blockquote> <ul class="simple"> <li>ОбъектПерехватчикГК = СоздатьОбъект("КлассПерехватчикГК");</li> <li>Перехватчик = СоздатьОбъект("Перехватчик");</li> <li>Перехватчик. <a class="reference" href="#id5">ПерехватитьСобытияГК</a> (КонтФормы, ОбъектПерехватчикГК);</li> </ul> <p>или</p> <ul class="simple"> <li>Перехватчик. <a class="reference" href="#id10">ПерехватитьСобытияГлобальногоМодуля</a> (ОбъектПерехватчикГМ);</li> </ul> <p>или</p> <ul class="simple"> <li>ПредОбработчик = Перехватчик. <a class="reference" href="#id5">ПерехватитьСобытияГК</a> (ОбъектПерехватчикГК);</li> <li>ПредОбработчик = Перехватчик. <a class="reference" href="#id10">ПерехватитьСобытияГлобальногоМодуля</a> (ОбъектПерехватчикГМ);</li> </ul> </blockquote> <p>4. В перехваченном контексте или перехваченном глобальном модуле <strong>обработчики событий могут вообще отсутствовать или быть определены без атрибута Экспорт</strong>.</p> <p>5. После установки перехвата при вызове системой события управление будет передано в соответствующий обработчик события в классе-перехватчике. В этом случае встроенные обработчики предопределенных событий в перехваченном контексте или глобальном модуле автоматически вызываться не будут. Разработчик сам должен вызвать данные обработчики (см. п.7), когда это необходимо.</p> <p>6. Для включения перехвата событий глобального модуля необходимо использовать код</p> <blockquote> <ul class="simple"> <li>ОбъектПерехватчикСобытийГлобальногоМодуля = СоздатьОбъект("КлассПерехватчикСобытийГлобальногоМодуля");</li> <li>Перехватчик = СоздатьОбъект("Перехватчик");</li> <li>Перехватчик. <a class="reference" href="#id10">ПерехватитьСобытияГлобальногоМодуля</a> (ОбъектПерехватчикСобытийГлобальногоМодуля);</li> </ul> </blockquote> <p>7. При необходимости вызова обработчиков событий, которые определены в контексте или в глобальном модуле, в обработчике события внутри пользовательского модуля нужно использовать код</p> <blockquote> <p>Процедура Событие_ПриЗакрытии() Экспорт</p> <ul class="simple"> <li>Перехватчик = СоздатьОбъект("Перехватчик");</li> <li>Перехватчик. <a class="reference" href="#id7">ВыполнитьОригинальноеСобытиеГК</a> (КонтПерехваченнойФормы, "ПриЗакрытии");</li> </ul> <p>или для вызова предыдущего сохраненного обработчика</p> <ul class="simple"> <li>Перехватчик. <a class="reference" href="#id6">ВыполнитьСобытиеГК</a> (КонтПерехваченнойФормы, "ПриЗакрытии", ПредыдущийОбработчик);</li> </ul> <p>КонецПроцедуры</p> </blockquote> <p>или</p> <blockquote> <p>Процедура СобытиеГМ_ПриЗавершенииРаботыСистемы() Экспорт</p> <ul class="simple"> <li>Перехватчик = СоздатьОбъект("Перехватчик");</li> <li>Перехватчик. <a class="reference" href="#id12">ВыполнитьОригинальноеСобытиеГлобальногоМодуля</a> ("ПриЗавершенииРаботыСистемы");</li> </ul> <p>или для вызова предыдущего сохраненного обработчика</p> <ul class="simple"> <li>Перехватчик. <a class="reference" href="#id11">ВыполнитьСобытиеГлобальногоМодуля</a> ("ПриЗавершенииРаботыСистемы", ПредыдущийОбработчик);</li> </ul> <p>КонецПроцедуры</p> </blockquote> <p>При этом неважно, существует ли обработчик в модуле формы или глобальном модуле.</p> <p><strong>Таким образом, для обработки цепочки из нескольких перехватчиков существуют методы</strong> <a class="reference" href="#id6">ВыполнитьСобытиеГК</a> <a class="reference" href="#id11">ВыполнитьСобытиеГлобальногоМодуля</a></p> <p>8. <strong>Важно помнить, что после установки перехвата на контекст формы или глобальный модуль на пользовательский объект-перехватчик добавляется еще одна ссылка.</strong> В результате деструктор данного объекта-перехватчика будет вызван только при закрытии формы или завершении работы 1С соответственно или снятии перехвата вручную.</p> <p>Например, код</p> <blockquote> <ul class="simple"> <li>ОбъектПерехватчикГК = СоздатьОбъект("КлассПерехватчикГК");</li> <li>Перехватчик = СоздатьОбъект("Перехватчик");</li> <li>Перехватчик. <a class="reference" href="#id5">ПерехватитьСобытияГК</a> (КонтФормы, ОбъектПерехватчикГК);</li> <li>ОбъектПерехватчикГК = 0;</li> </ul> </blockquote> <p>не вызывет деструктор немедленно!</p> <p>9. <strong>Метод-обработчик события может возвращать необходимый статус возврата.</strong> Для этого обработчик нужно определить как функцию</p> <pre class="literal-block"> Функция ПриЗакрытии() Экспорт // текст обработчика события Возврат 0; // т.е. фактически СтатусВозврата(0) и форма не будет закрыта.
