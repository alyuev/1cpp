# Восстановление имён тест-конфигурации

В исходном архиве `icpp source.zip` (и в git-снимке) у тест-конфигурации `Testing/`
были **испорчены кириллические имена файлов и папок** — они превратились в
последовательности символа-replacement `U+FFFD` (`?`). Порча возникла ещё до
упаковки архива (при переносе/распаковке с потерей кодировки cp1251); содержимое
файлов при этом уцелело.

## Как восстановлено

Правильные имена взяты из **уцелевших CVS-метаданных** самого репозитория:

- имена **папок** — из `CVS/Repository` (полный путь категории → базовое имя),
  восстановлены однозначно на 100%;
- имена **файлов** — из `CVS/Entries` по паре (длина имени, расширение); где в одной
  папке несколько файлов с одинаковой (длина, расширение) — доразобрано по
  различающим токенам имени в содержимом модуля.

Дополнительно:

- служебные каталоги `CVS/` **удалены** (мёртвая метадата древнего CVS, в git не
  нужна; именно она давала конфликты при слиянии дублей);
- **папки-дубли** (для 14 категорий из-за кривой распаковки была вторая копия с
  `?`-именем) слиты; идентичные файлы схлопнуты, данные не потеряны.

## Что осталось разобрать вручную

Небольшой хвост тест-файлов, где содержимое НЕ содержит различающих токенов имени
(несколько `.txt`/`.lst` в `UnitTests/OOP` и `UnitTests/SpecialClasses`):

- **14 файлов** оставлены с `?`-именем;
- **7 файлов** сохранены как `*.CONFLICT2.*` (обе версии на месте — контент разный).

Полный список — в `RECONSTRUCTION_flagged.txt`. Правильные имена-кандидаты для них
известны (из `CVS/Entries`), но сопоставить конкретный файл с именем автоматически
не удалось — нужно сверить по содержимому вручную либо взять из чистого checkout'а
версии 3.2.4.3, если найдётся.
